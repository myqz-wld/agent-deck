## 用法（给 Claude Code 的内部约定）

收到用户「纠正性 / 偏好性」反馈时（典型信号：「不要…」「应该…」「我已经说过…」「以后…」「记住…」「每次…」），按下面流程操作：

1. **判断范围**：只对**项目内的工程偏好 / 设计取舍 / 工作流偏好**计数。一次性请求（"帮我改这个 bug"）不算。
2. **读本文件**，找语义相近的已有条目：
   - 找到 → `count` 字段 +1，更新 `last_at` 为今天日期
   - 没找到 → 新增条目，`count: 1`
3. **count 达到 3** → **主动**告诉用户："这条偏好累计 3 次了，建议升级到 CLAUDE.md 项目约定，要做吗？" 等用户确认后：
   - 把条目重写成正式约定写入 [CLAUDE.md](../CLAUDE.md) 「项目特定约定」最贴合的小节（或新建一个）
   - 从本文件删除该条目
4. **count < 3** → 静默更新本文件，不打扰用户
