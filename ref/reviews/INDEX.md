# Reviews Index

> Periodic or triggered debug, code review, performance audit, and security review reports. Functional changes belong in [`ref/changelogs/`](../changelogs/INDEX.md); this directory focuses on fixes and hardening.

## Naming

Use `REVIEW_X.md`, where `X` is the next integer and historically aligns with `CHANGELOG_X.md`. Before creating a new file, run `ls ref/reviews/` and find the current maximum `X`.

## File Structure

- Trigger context (user request, periodic review, before a large refactor, and similar cases).
- Method (reviewer pairing, scope, and tools).
- Decision list (`fixed / rebutted / unknown`) with evidence (`file:line` plus a code snippet).
- Fix entries grouped by severity.
- Related changelog entry for fixes landed from the review.
