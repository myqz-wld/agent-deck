# CHANGELOG_344: Restore immediate Claude streaming input scheduling

## Summary

Claude Code adapter user messages sent through the SDK streaming input queue are now explicitly marked with immediate priority. This restores mid-turn input scheduling after the Claude Agent SDK introduced message priority metadata that otherwise lets messages fall through to next-turn consumption.

## Changes

- Added `priority: 'now'` to Claude SDK `SDKUserMessage` payloads generated by `StreamProcessor.makeUserMessage`.
- Covered both plain-text and attachment-backed Claude user messages with focused regression tests.

## Root Cause

The Agent Deck bridge was still waking the SDK input stream immediately: `sendMessage` pushed into `pendingUserMessages` and called `notify`, and the SDK `streamInput` implementation writes each yielded JSONL user message to the Claude Code subprocess immediately. The missing piece was the new Claude SDK/CLI scheduling metadata: without `priority: 'now'`, a streamed user message can be treated as the next user turn instead of being considered during the active turn.
